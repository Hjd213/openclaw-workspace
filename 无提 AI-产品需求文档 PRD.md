# 无提 AI 产品需求文档 (PRD)

> **产品名称**：无提 AI（WuTi AI）  
> **产品定位**：基于 Glass 二次开发的多模态桌面助手  
> **文档版本**：v1.0  
> **更新日期**：2026-04-11  
> **产品状态**：需求定义阶段

---

## 1. 产品概述

### 1.1 产品愿景

**打造"AI 主动找人"的桌面智能助手** —— 从传统的"用户找 AI"（需要手动输入 prompt）升级为"AI 主动找人"（基于上下文主动提供智能建议），降低用户使用 AI 的门槛，提升工作效率。

### 1.2 产品背景

**目标场景**：
- 面试辅助
- 商业会谈
- 医疗辅助
- 日常办公场景

**核心痛点**：
1. 用户需要手写 prompt，使用门槛高
2. 现有工具被动响应，无法主动感知工作场景
3. 多模态信息（屏幕 + 语音）无法统一理解
4. 行业场景（医疗、金融）缺乏专业化支持

### 1.3 目标与成功指标

| 指标类型 | 指标名称 | 目标值 |
|---------|---------|-------|
| **业务目标** | 积水潭医院部署完成率 | 100% |
| **用户体验目标** | 主动建议准确率 | ≥80% |
| **技术指标** | 响应延迟 | ≤200ms（端侧推理） |
| **技术指标** | 离线可用性 | 支持核心功能离线使用 |

### 1.4 范围与边界

**包含的功能**：
- ✅ 全局桌面上下文感知（屏幕 + 语音 + 操作行为）
- ✅ 实时语音转录
- ✅ 主动场景意图识别
- ✅ 多模态交互入口（语音/文本/快捷键）
- ✅ 场景模板化智能建议（医疗/金融/销售）
- ✅ 实时上下文语义理解
- ✅ 多模态会议内容自动整理
- ✅ 简易自动化执行（文件整理/格式转换/文本提取）

**不包含的功能**：
- ❌ 复杂的跨应用业务操作自动化
- ❌ 需要高权限的系统级控制
- ❌ 需要人工审核的高风险操作

## 4. 详细功能说明

### 4.1 多模态感知与主动交互模块

#### 4.1.1 全局桌面上下文感知（原始数据采集）

**功能描述**：
通过系统级读屏技术实时采集全桌面屏幕内容，同时支持音频输入采集、用户操作行为轨迹记录，全面获取用户当前的工作上下文，支持跨应用、跨窗口的全局内容感知，同时支持用户手动上传内容。

**前置条件**：
- 用户已授权屏幕读取权限
- 用户已授权麦克风权限（可选）

**业务流程**：
1. 用户启动无提 AI
2. 系统请求必要的系统权限（屏幕录制、麦克风）
3. 用户授权后，系统开始实时采集：
   - 屏幕内容（OCR 识别文字）
   - 音频输入（语音转录）
   - 用户操作行为（点击、滚动、输入等）
4. 采集的数据经过本地预处理后，存储为结构化上下文

**异常流程**：
- 权限被拒绝 → 提示用户手动授权，或降级为手动上传模式
- 屏幕内容无法识别 → 提示用户检查显示设置或尝试手动上传

**验收标准**：
- [ ] 支持 Windows/macOS 系统级屏幕读取
- [ ] 支持跨应用、跨窗口的内容感知
- [ ] 支持用户手动上传文件（PDF、Word、Excel、图片）
- [ ] 采集延迟 ≤500ms

**备注/约束**：
- 应当考虑用户桌面只展示文档或 PPT 的一部分时，是否读取整个文件
- 文字和语音支持多语言（中文、英文优先）
- 可手动开启/关闭感知功能

---

#### 4.1.2 语音转录

**功能描述**：
支持类 Glass 的实时语音转录能力，支持会议、沟通等多场景下的实时语音转文本，转录内容可自动生成文本记录，同时转录结果会作为多模态上下文的一部分，辅助 AI 更精准地识别用户工作场景。

**前置条件**：
- 用户已授权麦克风权限
- 已配置语音识别服务（本地 Whisper 或云端 STT）

**业务流程**：
1. 用户开启语音转录功能
2. 系统实时采集音频流
3. 音频流经过 STT 引擎转换为文本
4. 转录文本带时间戳存储，支持后续检索
5. 转录结果作为上下文输入给 AI 理解模块

**异常流程**：
- 环境噪音过大 → 提示用户检查麦克风或降低噪音
- 网络中断（云端 STT）→ 自动切换至本地 Whisper（如有）

**验收标准**：
- [ ] 支持实时转录（延迟 ≤2 秒）
- [ ] 支持多语言识别（中文、英文）
- [ ] 转录准确率 ≥85%（安静环境）
- [ ] 支持说话人区分（可选）

**备注**：
- 多语言支持
- 支持离线模式（本地 Whisper）

---

#### 4.1.3 主动场景意图识别

**功能描述**：
基于多模态上下文，自动识别用户的工作场景与潜在需求，无需用户主动唤醒助手，即可主动触发智能建议。

**前置条件**：
- 全局上下文感知已启用
- 用户已开启"主动建议"功能

**业务流程**：
1. 系统持续分析用户当前上下文（屏幕内容 + 语音 + 行为）
2. 意图识别模型判断用户是否需要帮助
3. 如识别到潜在需求，生成智能建议
4. 以非侵入式弹窗形式展示建议
5. 用户可选择采纳、忽略或关闭主动建议

**异常流程**：
- 误判用户意图 → 用户可关闭主动建议，系统学习反馈
- 建议不准确 → 提供"反馈"入口，优化模型

**验收标准**：
- [ ] 支持常见场景识别（会议、文档编辑、浏览网页、沟通）
- [ ] 主动建议触发准确率 ≥70%
- [ ] 用户可完全关闭主动建议功能
- [ ] 弹窗不干扰用户当前操作

**备注**：
- 实现 AI 找人的逆逻辑，避免手写 prompt 困难的痛点
- 需要平衡"主动"与"打扰"的边界

---

#### 4.1.4 多模态交互入口

**功能描述**：
支持语音、文本、快捷键唤醒等多种交互方式，用户可随时主动召唤助手，也可接收助手的主动建议弹窗，实现自然流畅的人机交互。

**前置条件**：
- 应用已安装并启动

**业务流程**：
1. 用户通过以下方式之一唤醒助手：
   - 快捷键（如 Ctrl+Enter）
   - 语音唤醒词（如"嘿，无提"）
   - 点击系统托盘图标
   - 接收主动建议弹窗
2. 交互窗口弹出，展示对话界面
3. 用户输入问题或选择建议
4. 系统返回结果

**异常流程**：
- 快捷键冲突 → 允许用户自定义快捷键
- 语音唤醒不灵敏 → 提供灵敏度调节

**验收标准**：
- [ ] 支持至少 3 种唤醒方式
- [ ] 唤醒响应时间 ≤500ms
- [ ] 交互窗口支持拖拽、缩放
- [ ] 支持录屏时隐藏交互窗口（可选）

**备注**：
- 交互窗口可选在录屏时是否可见
- 支持手动开启/关闭主动建议功能

---

### 4.2 智能决策与场景化功能模块

#### 4.2.1 场景模板化智能建议

**功能描述**：
内置医疗、金融等专属场景模板，结合用户当前工作上下文，生成符合行业规范的智能建议。

**前置条件**：
- 用户已选择或系统已识别当前场景
- 上下文感知已启用

**业务流程**：
1. 系统识别或用户选择场景模板（医疗/金融/销售）
2. 结合当前上下文（屏幕内容 + 语音）
3. 调用对应场景的 AI 模型
4. 生成符合行业规范的建议
5. 展示建议并支持用户反馈

**异常流程**：
- 场景识别错误 → 用户可手动切换场景模板
- 建议不符合规范 → 提供反馈入口，持续优化

**验收标准**：
- [ ] 支持至少 3 个行业场景（医疗、金融、销售）
- [ ] 每个场景有专属的 prompt 模板
- [ ] 建议内容符合行业规范（需专家审核）
- [ ] 支持用户自定义场景模板

**示例场景**：
- **医疗场景**：病历分析、诊断建议、药品相互作用检查
- **金融场景**：风险提示、竞品对比、合规性检查
- **销售场景**：话术辅助、客户意图识别、跟进建议

---

#### 4.2.2 实时上下文语义理解（语义处理）

**功能描述**：
基于多模态大模型，对用户当前的屏幕、语音内容进行实时语义理解，同时支持 PDF、Excel、Word 等各类办公文档的语义理解，用户可直接针对当前工作界面提问。

**前置条件**：
- 上下文感知已启用
- 已配置大模型服务（本地或云端）

**业务流程**：
1. 用户针对当前界面提问（语音或文本）
2. 系统结合当前屏幕内容 + 语音上下文 + 文档内容
3. 调用多模态大模型进行理解
4. 生成针对性的回答
5. 支持追问和跨文档关联分析

**异常流程**：
- 模型响应超时 → 提示用户或切换至轻量模型
- 云端服务不可用 → 自动切换至本地模型（如有）

**验收标准**：
- [ ] 支持针对当前屏幕内容提问
- [ ] 支持 PDF、Excel、Word 文档的语义理解
- [ ] 支持跨文档关联分析
- [ ] 响应时间 ≤3 秒（云端），≤10 秒（本地）
- [ ] 支持离线使用（本地模型）

**备注**：
- 同时支持本地与云端模型
- 云端模型适配主流模型（ChatGPT、Gemini、Qwen、豆包等）
- 离线可用
- 跨文档关联分析

---

#### 4.2.3 多模态会议内容自动整理

**功能描述**：
结合语音转录内容与屏幕内容，自动完成会议内容的整理，自动提取会议要点、决策事项、待办任务，生成结构化的会议纪要。

**前置条件**：
- 语音转录已启用
- 屏幕内容感知已启用

**业务流程**：
1. 用户开启"会议模式"
2. 系统实时采集语音转录 + 屏幕内容
3. 会议结束后，自动整理内容
4. 提取：
   - 会议要点
   - 决策事项
   - 待办任务（Action Items）
   - 参会人员（如可识别）
5. 生成结构化会议纪要，支持导出

**异常流程**：
- 会议中途暂停 → 支持分段整理
- 识别错误 → 支持手动编辑纪要

**验收标准**：
- [ ] 支持自动提取会议要点
- [ ] 支持自动识别待办任务
- [ ] 支持导出为 Word/Markdown/飞书文档
- [ ] 支持按会议时间检索历史纪要

---

#### 4.2.4 简易自动化执行

**功能描述**：
仅支持极简执行操作，例如文件分类整理、文档格式转换、纯文本内容提取等，不涉及复杂的跨应用业务操作，且需要用户确认执行，保障执行低出错。

**前置条件**：
- 用户已授权文件操作权限
- 用户已确认执行操作

**业务流程**：
1. 用户选择要执行的自动化任务
2. 系统展示任务预览和影响范围
3. 用户确认执行
4. 系统执行任务
5. 展示执行结果

**异常流程**：
- 执行失败 → 回滚操作，提示错误原因
- 用户中途取消 → 停止执行，保留原状

**验收标准**：
- [ ] 支持文件分类整理
- [ ] 支持文档格式转换（PDF↔Word↔Excel）
- [ ] 支持纯文本内容提取
- [ ] 所有操作需用户确认
- [ ] 支持操作撤销/回滚

**备注**：
- 仅支持最简自动化，避免高成本误操作与失控
- 不涉及复杂的跨应用业务操作

## 5. 非功能需求

### 5.1 性能要求

| 指标 | 要求 | 备注 |
|-----|------|------|
| 屏幕采集延迟 | ≤500ms | 本地处理 |
| 语音转录延迟 | ≤2 秒 | 实时转录 |
| AI 响应时间 | ≤3 秒（云端）/ ≤10 秒（本地） | 多模态理解 |
| 主动建议触发延迟 | ≤1 秒 | 场景识别后 |
| 内存占用 | ≤500MB | 空闲状态 |
| CPU 占用 | ≤10% | 空闲状态 |

### 5.2 安全要求

| 要求 | 说明 |
|-----|------|
| 数据本地化 | 敏感数据（屏幕内容、语音）优先本地处理 |
| 加密存储 | 本地存储的数据需加密 |
| 权限最小化 | 仅请求必要的系统权限 |
| 用户可控 | 所有感知功能可手动开启/关闭 |
| 隐私保护 | 支持录屏时隐藏交互窗口 |
| 合规性 | 符合医疗/金融等行业数据合规要求 |

### 5.3 兼容性要求

| 平台 | 支持版本 |
|-----|---------|
| Windows | Windows 10/11（64 位） |
| macOS | macOS 12+（Intel/Apple Silicon） |
| 浏览器 | Chrome 80+, Edge 80+（Web 版） |

## 6. 数据需求

**核心指标**：
- 日活跃用户数（DAU）
- 主动建议采纳率
- AI 查询平均响应时间
- 语音转录准确率
- 会议纪要生成数量
