# 自动备份配置说明

> 让 OpenClaw 自动把你的配置推送到 GitHub 🐾

---

## 方案 A: 使用心跳机制（推荐）

**已经配置好了！** 每次 OpenClaw 心跳时会自动检查并备份。

**心跳频率：** 默认每天 2-3 次（由 OpenClaw 控制）

**工作原理：**
1. OpenClaw 定期触发心跳
2. 助手读取 `HEARTBEAT.md` 中的任务
3. 检查 workspace 是否有更改
4. 自动执行 `git add` → `git commit` → `git push`

**无需额外配置！** 🎉

---

## 方案 B: Windows 任务计划程序（更精确）

如果你想精确控制备份时间（比如每天凌晨 2 点）：

### 步骤 1: 打开任务计划程序

```powershell
taskschd.msc
```

### 步骤 2: 创建基本任务

1. 点 **创建基本任务**
2. 名称：`OpenClaw 自动备份`
3. 触发器：选 **每天**
4. 时间：选一个你不常用电脑的时间（如凌晨 2:00）
5. 操作：选 **启动程序**
6. 程序/脚本：`powershell.exe`
7. 添加参数：
   ```
   -ExecutionPolicy Bypass -File "C:\Users\你的用户名\.openclaw\workspace\auto-backup.ps1"
   ```
8. 完成

### 步骤 3: 测试

右键任务 → **运行**，然后检查 GitHub 是否有更新。

---

## 方案 C: 手动触发

随时可以手动备份：

```powershell
cd C:\Users\你的用户名\.openclaw\workspace
git add .
git commit -m "手动备份"
git push
```

---

## 注意事项

1. **网络问题** - 如果推送失败，脚本会记录状态，下次再试
2. **敏感信息** - `.gitignore` 已配置，不会上传 API Key 等敏感内容
3. **冲突处理** - 如果多设备同时修改，可能需要手动解决冲突

---

## 查看备份状态

```powershell
# 查看最近的提交
cd C:\Users\你的用户名\.openclaw\workspace
git log --oneline -5

# 查看是否有未推送的更改
git status
```

---

## 禁用自动备份

如果不想自动备份了：

1. 编辑 `HEARTBEAT.md`
2. 删除或注释掉自动备份任务部分
3. 或者删除 Windows 任务计划程序中的任务

---

**完成！** 现在你的配置会自动备份到 GitHub 了 🎉
